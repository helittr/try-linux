# 安装依赖

在使用镜像之前你需要安装一些必备的程序，dockeropen in new window 是一个工具，可以使程序运行的与主机隔离的环境中。

在此之前你可能使用过想 Vmware, VirtualBox, qemu 等，虚拟机程序。他们可以运行一个独立的操作系统,如: windows, debian, ubuntu …… 。 他们的都是通过代码虚拟化 CPU, 硬盘等硬件设备, 通过软件的方式模拟一个机器码执行环境，来运行操作系统。通常这些虚拟机产品的性能都比较差, 因为大部分计算都用于模拟 CPU 等硬件设备，并且操作系统镜像通常很大难以分发。

而 docker 与这个虚拟机产品相比，它不需要模拟硬件设备，而是使用宿主机的系统，着重在隔离进程, 文件, 等资源。 而不是隔离整个操作系统， 所以有着与在宿主机几乎一样的性能。被广泛运用在分布式，云应用，自动话部署等领域。

它也可以用与开发中，打包一个源码和构建环境的镜像，省去了环境搭建的时间并且使所有的开发者都使用相同的环境，可以避免因环境而导致的各种问题。

# 运行容器

docker run -d -p 22:22  -e ROOTPASSWD=root --name try try-linux
  • -d 选项表示容器启动后在后台运行，如果不指定该选项，关闭命令行或者使用Ctrl+C终止进程，容器也会同时停止运行。
  • -p 将容器中的端口映射到主机中，':' 前面是指 host 端口, 后面数字是指容器中的端口。映射后主机的端口的数据将会被转发到容器中的端口。22 是 sshd 服务默认的端口号。请勿修改容器中的端口，如果你主机的端口 22 已经被占用你可以使用其他端口，比如:-p 222:22。
  • -e 指定容器的环境变量，容器启动时会读取 环境变量 ROOTPASSWD 的值来设定 root 的用户密码。
  • --name 指定容器的名字。
  • try-linux 用于启动容器的镜像的名字。 你可以通过设置环境变量 ROOTPASSWD， 来设置容器中 root 用户的密码，默认为 root.

# 启动现有容器

容器启动后，所有数据将保存在容器中，如果你需要继续使用请不要删除该容器。当你的电脑重启后，你的容器并不会自动启动，也不需要创建一个新的容器。你可以通过下面命令来重新运行已经创建的容器。

docker start try
  • try 是你在 运行容器时通过 --name 指定的容器名字。

# 测试连接 Test

如果你使用的是 bash shell 执行下面命令来连接到容器：

ssh root@`hostname`.local

ssh root@$(hostname).local

根据提示输入 root 用户密码 root, 你输入密码时终端将不会显示，输入后按 Enter确认，连接成功后你将进入容器中的命令行。

特别注意

如果你在启动容器时 使用 -p 修改了默认的 22 端口号，在连接时也必须通过 ssh 命令的 -p 选项来指定连接端口。 如:

ssh -p 222 root@`hostname`.local

ssh -p 222 root@$(hostname).local

# 设置免密登录

  1. 如果你的系统中没有可用的密钥对,你可以使用 ssh-keygen 生成 ssh 密钥对。
ssh-keygen -t rsa

所有输入信息留空，直到设置完成。生成的密钥对将保存在 ~/.ssh/ 文件夹中。

  1. 将公钥上传至容器

对于 bash 用户, 你可以使用 ssh-copy-id 命令上传公钥至主机。如果你是 Windows 用户你可以安装 Gitopen in new window 客户端，来获得一个 git bash 终端。请参考 安装 git。

ssh-copy-id root@$(hostname).local

ssh  root@$(hostname).local "mkdir ~/.ssh/; echo $(cat ~/.ssh/id_rsa.pub) >> ~/.ssh/authorized_keys"

配置成功后你可以无需输入密码，便可以登录到主机。如何登录可以参考 测试连接。
